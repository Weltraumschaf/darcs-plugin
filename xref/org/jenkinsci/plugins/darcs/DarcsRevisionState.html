14  /**
15   *
16   * Feedback from mailing list:
17   * stephen.alan.connolly@gmail.com:
18   * <quote>
19   * I think you would be better served by computing the sha1 or md5 of all the hashes as strings.  Relying on Collections.hashCode is dangerous.  Relying on String.hashCode, which is just:
20   *
21   *     public int hashCode() {
22   *         int h = hash;
23   *         if (h == 0) {
24   *             int off = offset;
25   *             char val[] = value;
26   *             int len = count;
27   *
28   *             for (int i = 0; i < len; i++) {
29   *                 h = 31*h + val[off++];
30   *             }
31   *             hash = h;
32   *         }
33   *         return h;
34   *     }
35   *
36   * Is going to be a tad weak given that all the characters are from the set [0-9a-f-] i.e. there are only 17 out of 255.
37   *
38   * The List.hashCode speci is
39   *     public int hashCode() {
40   *         int hashCode = 1;
41   *         Iterator<E> i = iterator();
42   *         while (i.hasNext()) {
43   *             E obj = i.next();
44   *             hashCode = 31*hashCode + (obj==null ? 0 : obj.hashCode());
45   *         }
46   *         return hashCode;
47   *     }
48   *
49   * Also I would recommend sorting the lists before hashing them.
50   *
51   * However, if you are looking for a short path to saying there is a difference, the Collections.sort(list1); Collections.sort(list2); if (list1.hashCode() != list2.hashCode()) check should be OK...
52   *
53   * Just remember that list1.hashCode() == list2.hashCode() does not in anyway claim that there is no change, so you will have to go down the long path anyway.
54   * </quote>
55   *
56   * kkawaguchi@cloudbees.com:
57   * <quote>
58   * There's utility code in Jenkins that computes MD5 digest of arbitrary byte stream or string. I think that seems like a good and cheap enough hashing for this kind of purpose.
59   *
60   * Javadoc
61   * hudson.Util.getDigestOf()
62   * public static String getDigestOf(InputStream source)
63   *                          throws IOException
64   *
65   *    Computes MD5 digest of the given input stream.
66   * </quote>
67   *
68   * @author Sven Strittmatter <ich@weltraumschaf.de>
69   */
70  public class DarcsRevisionState extends SCMRevisionState {
71  
72      private final DarcsChangeSetList changes;
73  
74      public DarcsRevisionState(DarcsChangeSetList changes) {
75          super();
76          this.changes = changes;
77      }
78  
79      public DarcsChangeSetList getChanges() {
80          return changes;
81      }
82  
83      @Override
84      public String toString() {
85          return getChanges().digest();
86      }
87  
88      @Override
89      public boolean equals(Object other) {
90          boolean result = false;
91  
92          if (other instanceof DarcsRevisionState) {
93              DarcsRevisionState that = (DarcsRevisionState) other;
94              return getChanges().equals(that.getChanges());
95          }
96  
97          return result;
98      }
99  
100     @Override
101     public int hashCode() {
102         return changes.hashCode();
103     }
104 }
